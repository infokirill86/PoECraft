# Separate Gates, Risks, and Stop Conditions

## Separately gated

- D3-D5 acceptance and Reveal runtime.
- Echoes and Ancient-MML/constraint persistence.
- Omen of Light runtime.
- Putrefaction, multiple placeholders, and corrupt-state mechanics.
- Astrid/crafted-capacity and repeat/replacement Essence semantics.
- Revealed-Desecrated Fracture and PD-013.
- Cyclic routes, compact retry syntax, route generation, search, comparison, ranking, costs, economics, EV, or advice.
- Public numeric release, automation, and SOURCE/PROVENANCE/MML/crafted-capacity closure.

## Main risks

| Risk | Control |
|---|---|
| Evaluator becomes planner | Caller supplies the complete graph; runtime never creates/compares nodes or actions |
| Predicate language becomes arbitrary code | Small named registry only; no expression language or callbacks |
| Conditional branch uses stale root state | Resolve and classify from each actual branch state; digest in trace |
| Retry becomes unbounded | DAG only; explicit unrolling and root-to-leaf operation cap |
| Exact result silently approximated | Structured ceiling stop; MC is separate explicit call |
| New operation slips through route schema | Resolver admission plus explicit executor registry at every node |
| Success classifier invents truth | Interpret only accepted success criteria; unsupported shapes fail closed |
| Internal results become advice | No costs/ranking/recommendation fields; public numeric release remains closed |

## Hard stop triggers for later M48-A

- Need to alter accepted operation mechanics or admission.
- Need to add a new success rule rather than interpret accepted config.
- Any route generation, optimization, recommendation, cost, EV, or ranking behavior.
- Cycle, unbounded retry, hidden approximation, or incomplete branch coverage.
- Public numeric release or boundary closure.
