# Current Capability and Missing-Capability Map

## What P2C can do now

- Build and mutate Normal, Magic, and Rare quarterstaff states.
- Execute accepted rarity-progression currencies, Exalted/Chaos variants, Annulment, Alchemy, Greater/Perfect Essences, Fracture, and three Jawbones.
- Apply the accepted ten-Omen independent modifier layer.
- Resolve each operation fail-closed from runtime admission metadata.
- Evaluate caller-supplied fixed sequences of one to eight operations.
- Re-resolve every later step from the actual branch state.
- Enumerate exact rational paths under ceilings and run deterministic seeded MC/replay.
- Record a hidden Desecrated placeholder, but not Reveal it.

## What is missing

| Capability | Current state | Consequence |
|---|---|---|
| Executable target/success classifier | `config/success_criteria.yaml` is validated data, but no current runtime classifier/evidence suite was found | A result cannot be routed automatically to TOP/ACCEPTABLE/NOT_SUCCESS inside an evaluator |
| Conditional route evaluation | M43-A accepts only a fixed linear tuple | Real user-authored fallback/stop branches cannot be evaluated |
| Reveal | D3-D5 remain proposed | Jawbone route intentionally stops at hidden placeholder |
| Echoes | Depends on Reveal sampler and Ancient-MML conflict | Cannot be admitted independently |
| Crafted-capacity expansion | Astrid catalog contour exists; interaction contract remains open | Double-crafted/Essence routes remain blocked |
| Revealed Desecrated + Fracture | PD-013 remains open | Combined state cannot enter accepted runtime |
| Strategy search/advice | Intentionally closed | P2C evaluates submitted actions but does not find or recommend them |

## Product gap

The simulator has many operation executors but still behaves like a fixed script runner. A bounded branching evaluator makes those accepted operations usable as a real, user-written crafting process while keeping strategy choice outside the engine.
