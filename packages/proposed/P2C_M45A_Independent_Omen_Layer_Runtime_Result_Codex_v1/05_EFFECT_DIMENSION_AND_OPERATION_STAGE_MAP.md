# Effect-Dimension and Operation-Stage Map

| Effect | Load-bearing stage | Preserved downstream behavior |
|---|---|---|
| Exaltation side | accepted legal weighted add pool before each affected draw | capacity, family/group conflicts, weights, MML, canonical validation |
| Annulment side | accepted fractured-protected removable-instance pool | uniform selection within filtered pool |
| Erasure side | accepted Chaos removal pool only | branch-specific post-removal add pool remains unchanged |
| Whittling | after ordinary eligibility and fractured exclusion | minimum modifier level, then uniform tie; accepted Chaos add unchanged |
| Crystallisation side | accepted Perfect Essence terminal-feasible removal pool | guaranteed Essence modifier and temporary crafted-count floor unchanged |
| Greater Exaltation count | accepted ordinary weighted add, repeated sequentially on a private state | legality and pool rebuilt after first add; atomic visibility |

The resolver produces one immutable single-operation plan. M43-A may evaluate that plan as one caller-supplied step, but M45-A adds no route-generation behavior.
