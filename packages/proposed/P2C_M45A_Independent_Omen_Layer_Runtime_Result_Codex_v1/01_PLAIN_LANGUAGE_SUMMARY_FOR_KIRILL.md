# Plain-Language Summary for Kirill

P2C can now, as a proposed audited layer, apply the selected Omens to currencies it already knows how to execute. The Omen does not replace the currency engine: it narrows the existing add/remove pool or changes the permitted add count, and then the existing executor continues.

Examples now covered internally include prefix-only Exalted additions, suffix-only Annulment removal, Chaos removal restricted by side or lowest modifier level, and Perfect Essence removal restricted by side. Greater Exaltation performs two ordinary weighted additions on a private working copy and exposes the result only if both succeed.

The safety boundary is explicit: seeing an Omen in YAML or enabling the Omen system does not make it executable. Each Omen needs both an accepted runtime-admission status and an accepted availability status. Only the ten rows authorized by the gate have those statuses.
