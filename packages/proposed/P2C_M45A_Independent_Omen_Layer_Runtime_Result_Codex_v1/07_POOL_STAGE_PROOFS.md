# Exalted, Annulment, Chaos, and Perfect-Essence Pool-Stage Proofs

Focused tests prove:

- Sinistral/Dextral Exaltation candidates all come from the requested side of the canonical ordinary-add pool.
- Sinistral/Dextral Annulment candidates all come from the requested side after fractured exclusion and remain uniformly weighted.
- Sinistral/Dextral Erasure changes only the Chaos removal pool. The rebuilt post-removal add pool records no inherited side filter.
- Sinistral/Dextral Crystallisation filters the already terminal-feasible Perfect Essence removal set. The guaranteed modifier ID and side remain the row-declared canonical output.
- Empty filtered pools preserve the input and return no-transition/no-consumption.

No parallel legality, capacity, removal, family/group, modifier-index, weight, canonical-state, exact-mass, or replay implementation was introduced.
