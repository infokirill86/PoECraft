# Deferred Scope and Boundaries

## Explicitly deferred

Deferred beyond M35 design:

- Annulment runtime implementation;
- Annulment acceptance as executable;
- Annulment selector variants using Omen side filters or desecrated-only filters;
- Chaos;
- Perfect Essence;
- Jawbone;
- Reveal;
- Lich-related behavior;
- Whittling / Erasure behavior;
- heterogeneous operation sequences;
- target success aggregation over Annulment;
- public numeric probability release;
- optimizer/advice/ranking;
- economics/EV/expected attempts;
- automation or GitHub Actions.

## Standing blockers remain open

This package does not close:

- SOURCE/PROVENANCE;
- MML;
- PD-013;
- server-truth uncertainty;
- public numeric release.

## What M35 may claim

M35 may claim only that:

- the project next-move direction is accepted;
- a design-only operation-admission framework is proposed;
- Annulment is the proposed first candidate design;
- M35-A may be proposed later as a separate implementation floor.

## What M35 must not claim

M35 must not claim that:

- Annulment runtime is accepted;
- Annulment is server-truth exact;
- Annulment probabilities are released publicly;
- existing data row `active_in_current_simulation: true` is sufficient for executable runtime acceptance;
- future operations can be admitted without separate gates.
