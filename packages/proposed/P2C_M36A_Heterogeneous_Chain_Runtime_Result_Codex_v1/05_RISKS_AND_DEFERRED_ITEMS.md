# Risks and Deferred Items

## Risks

- M36-A intentionally composes two accepted operations, so failures are most likely to appear at the chain boundary: state handoff, pool rebuild, path/terminal identity, and fail-closed admission checks.
- The local hook is effective only after a clone runs `git config core.hooksPath tools/hooks`. This is local Git behavior, not centralized GitHub enforcement.

## Deferred

- M36-A acceptance.
- Chains longer than two steps.
- Route planning.
- New operation admission.
- Chaos / Essence / Fracture / Desecrate / Jawbone / Reveal runtime.
- Annulment variants or omens.
- Optimizer, advice, ranking, economics, EV.
- Public numeric probability release.
- SOURCE/PROVENANCE closure.
- MML closure.
- PD-013 closure.
- GitHub Actions, watcher automation, supervised auto-run.

## Boundary statement

This package does not claim server truth and does not close any source/provenance blocker.
