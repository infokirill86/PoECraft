Perfect Seeking and Infinite do not exist in the prepared project inventory and were not invented.

## Decision 1: feasible removal pool

- Start from the accepted shared non-fractured removal pool.
- Retain only removal branches whose complete terminal state can install the canonical guaranteed modifier.
- Select uniformly from that feasible pool.
- If target side is full, only capacity-creating removals survive.
- Empty feasible pool fails before draw with no transition/no consumption.

## Decision 2: crafted precondition

- Require `crafted_count == 0` before pool construction.
- Any existing crafted modifier fails unchanged.
- Replacement, stacking, and repeat application remain source-open and unimplemented.
