# Crafted-capacity and repeat-application decision table

| Case | Evidence | Safe status for M42 |
|---|---|---|
| Item has no crafted modifier | Compatible with the official single-crafted limit | Candidate for M42-A after removal rule is selected |
| Item already has a Greater Essence crafted modifier | Current 0.5 rule allows only one crafted modifier | Do not allow or model replacement without explicit decision |
| Perfect Essence removes the existing crafted modifier and adds its own | Not stated by primary wording; community discussion conflicts | Unresolved; separate user decision/in-game verification |
| Existing crafted modifier is not selected for removal | New guaranteed crafted mod would exceed the current limit | Must fail or be inapplicable; no stacking |
| Second Perfect Essence on a Perfect-Essence item | Same replacement ambiguity | Unresolved; do not infer repeatability |
| Guaranteed family/group already present | Prepared project data says application is blocked | Keep as a necessary precondition, but it is not the whole crafted-capacity contract |

Recommended conservative M42-A precondition if adopted by a later gate: the input must contain zero crafted modifiers. This matches the current one-crafted-modifier rule and fails closed, but it deliberately does not claim that the game could never support replacement. A later verified replacement rule would be a separate expansion.
