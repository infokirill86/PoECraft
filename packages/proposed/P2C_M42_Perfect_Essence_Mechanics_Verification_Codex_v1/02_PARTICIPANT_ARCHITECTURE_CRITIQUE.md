# Participant and architecture critique

M42 is the right next broad wave. It adds remove-plus-guaranteed-add behavior that is directly useful to the quarterstaff simulator and reuses accepted kernels: canonical modifier resolution, fractured protection, removal path identity, atomic transitions, and terminal aggregation.

The clean base operation can and should be separated from Omens. Side Omens alter the removal filter; they are not part of base Perfect Essence semantics.

The requested batch is correctly sized at the design level, but runtime admission would be premature. Two unresolved rules affect exact path probabilities and even whether an item can be acted on. The better boundary is:

1. M42 verification and Claude audit;
2. explicit ChatGPT/User project-model decision on removal feasibility and crafted replacement;
3. one M42-A implementation batch for all six rows if the decision is complete.
