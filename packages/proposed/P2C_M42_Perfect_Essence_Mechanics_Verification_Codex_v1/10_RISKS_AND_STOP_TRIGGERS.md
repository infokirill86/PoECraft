# Risks and stop triggers

- Wrong removal conditioning produces exact probabilities for the wrong model.
- Treating current YAML as proof would silently accept an unresolved mechanic.
- Ignoring the current one-crafted-modifier rule would preserve an obsolete multi-Essence route.
- Treating community reports as primary authority would overstate confidence.
- Expanding the inventory from current external catalog data would bypass source reconciliation.

Stop before runtime if either removal feasibility or crafted replacement remains undecided. Stop on any operation/output/canonical-index mismatch, source conflict, unauthorized Omen/modifier behavior, runtime admission, public numeric output, optimizer/economics/advice, automation, or boundary closure.
