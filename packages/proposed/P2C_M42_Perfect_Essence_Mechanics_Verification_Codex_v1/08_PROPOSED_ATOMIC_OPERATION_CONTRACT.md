# Proposed atomic operation contract

Subject to a later explicit decision on the two open rules, the clean M42-A contract should be:

1. Resolve exactly one of the six admitted Perfect Essence rows through the accepted resolver.
2. Require Rare quarterstaff input and no unsupported variant/modifier layer.
3. Resolve the guaranteed modifier through operation data, essence output data, and the canonical modifier index.
4. Prevalidate item class, guaranteed family/group absence, crafted-capacity policy, and existence of at least one feasible non-fractured removal instance.
5. Select one removal-path identity uniformly from the adopted feasible pool.
6. Build an isolated post-removal state.
7. Install exactly the guaranteed modifier with no add-pool or weighted draw.
8. Validate the complete Rare terminal state through shared validators.
9. Commit removal plus guaranteed addition together.
10. Aggregate paths by canonical terminal-state identity; exact path mass remains distinct from terminal identity.

Any invalid precondition or branch failure returns no-transition/no-consumption with the original state unchanged. No remove-only terminal is permitted.
