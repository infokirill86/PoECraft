# Plain-Language Summary for Kirill

## What P2C can do now

The simulator now knows the basic currency path from a blank item to a Rare item, single add and remove operations, Chaos-like remove-and-add, Greater/Perfect variants, two Essence families, Alchemy, and fixed user-supplied sequences of accepted actions. It can calculate exact project-model outcomes where the state space remains bounded and can replay seeded simulations.

## What is missing most

The biggest practical gap is control over an accepted currency. Omens answer questions such as “add only a prefix”, “remove only a suffix”, or “make Chaos remove the lowest-level eligible modifier”. Without that layer, P2C can execute currencies but cannot represent several important control tools used around the intended physical-quarterstaff route.

## Recommendation

Make M45 the independent Omen modifier layer, not another chain-hardening wave and not Fracture yet. The active route already begins with a purchased fractured critical-strike staff, so implementing how that fracture was created would not improve the route currently being evaluated. More sequence infrastructure would also be premature infrastructure drift: the accepted evaluator already handles fixed sequences of one through eight actions.

The later M45-A implementation should admit only a pinned clean core of Omens that modify already accepted Exalted-like, Annulment, Chaos-like, and Perfect Essence behavior. It must remain a compiler of one currency plus modifiers, not a planner or recommendation system.

Several Omen records require separate treatment because the external catalogue changed or their probability/atomicity contract is not sufficiently specified. They are explicitly excluded rather than guessed.
