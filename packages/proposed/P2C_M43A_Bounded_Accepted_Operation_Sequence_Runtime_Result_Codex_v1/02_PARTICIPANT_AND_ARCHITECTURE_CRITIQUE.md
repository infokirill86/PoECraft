# Participant and Architecture Critique

The authorized boundary is right-sized.

- The accepted operation surface is now broad enough that composition gives more project value than another isolated currency.
- One coherent exact, MC, replay, registry, and diagnostic layer avoids partially usable sequence infrastructure.
- The sequence layer must remain an evaluator. Adding conditional policy, fallback, retry, route generation, or ranking would cross into a planner/optimizer gate.
- Exact enumeration and MC must share the accepted family harnesses and pool builders; the sequence layer must not copy game mechanics.

Implementation therefore uses an explicit currency-to-executor registry and resolves each live branch step through `OperationResolver`. Runtime admission alone is insufficient: an admitted row without a registered executor fails closed.

Alchemy remains deferred, not rejected.
