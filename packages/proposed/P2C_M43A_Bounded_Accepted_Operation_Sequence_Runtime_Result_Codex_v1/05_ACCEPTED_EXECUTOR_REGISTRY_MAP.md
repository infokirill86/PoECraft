# Accepted Executor Registry Map

The registry is explicit and immutable for a harness instance.

| Executor | Registered surface |
|---|---|
| `ordinary_add` | engine primitive plus admitted Greater/Perfect Exalted wrappers |
| `annulment` | base Annulment |
| `chaos_like` | base, Greater, and Perfect Chaos rows |
| `catalog_single_add` | admitted Transmutation, Augmentation, Regal variants and base Exalted |
| `greater_essence` | eight accepted Greater Essence rows |
| `perfect_essence` | six accepted Perfect Essence rows |

Evidence:

- registry keys exactly equal all currently runtime-admitted operation rows plus the `ordinary_add` engine primitive;
- every registered ID compiles through `OperationResolver` on a compatible state;
- removing a registered executor while its row remains admitted produces an explicit fail-closed error;
- Alchemy and other non-admitted operations remain blocked.

`active_in_current_simulation` is never used as execution authority.
