# Current and Missing Capability Map

## Accepted executable surface

| Capability | Current state |
|---|---|
| Ordinary weighted add | Accepted engine primitive |
| Rarity progression | Base/Greater/Perfect Transmutation, Augmentation, Regal |
| Exalted add | Base/Greater/Perfect Exalted |
| Removal | Base Annulment |
| Remove then add | Base/Greater/Perfect Chaos-like |
| MML filtering | Accepted interface and admitted row thresholds; broader MML remains open |
| Resolver | Accepted single-operation admission/compilation seam |
| Chains | Fixed two-step accepted `ordinary_add + Annulment` combinations only |
| Exact and seeded MC | Accepted for the admitted narrow lanes |

## Largest missing product capabilities

| Missing capability | Why it matters | Current blocker |
|---|---|---|
| Guaranteed specific modifier | Required for deterministic Essence crafting | No admitted forced/guaranteed-add operation |
| Perfect Essence replacement | Directly relevant to fractured Rare staff | Removal-side/capacity rule needs verification |
| Operation-modifying Omens | Directional control of accepted currencies | Repo catalogue is incomplete versus current external catalogue; multiple mechanics dimensions |
| Alchemy multi-add | Alternative item creation | Four sequential adds and discard semantics; lower relevance to primary fractured-Rare start |
| Fracturing | Creates protected start states | Primary route already starts fractured; revealed-Desecrated interaction disputed |
| Jawbone/Reveal | Desecrated crafting | Multi-stage placeholder/offer-set model and PD-013 boundary |
| Longer mixed chains | Route evaluation | More operation breadth is still higher value |
