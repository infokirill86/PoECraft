# Boundary Report

## Accepted/recorded

- M36 heterogeneous-chain design is recorded as accepted after Claude `GO WITH CHANGES` audit.
- Repo Integrity SHA Floor is authorized and implemented as minimal checksum tooling.

## Not accepted / not authorized

- M36-A implementation.
- Heterogeneous-chain runtime.
- Any new executable operation.
- Mechanics or data semantic changes.
- Optimizer, advice, ranking, economics, EV.
- Public numeric probability release.
- SOURCE/PROVENANCE closure.
- MML closure.
- PD-013 closure.
- GitHub Actions, supervised auto-run, or watcher automation.

## Why this floor is safe before M36-A

M36-A would add important runtime surface. The root checksum drift has already recurred, so fixing the repo-integrity mechanism first reduces audit noise before implementation resumes.
