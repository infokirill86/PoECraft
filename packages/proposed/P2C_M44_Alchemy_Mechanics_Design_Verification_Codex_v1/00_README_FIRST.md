# M44 Base Alchemy Mechanics Design Verification

Package type: `DESIGN_AND_MECHANICS_VERIFICATION`

Status: proposed. This package does not implement or admit Alchemy.

This package verifies the strongest available public wording for base Orb of Alchemy, separates confirmed behavior from project-model choices, and proposes a narrow atomic multi-add implementation floor for a later M44-A gate.

The main conclusion is intentionally conservative: non-fractured Normal/Magic input is a coherent M44-A core. Fractured Magic input remains fail-closed because the sources checked do not directly resolve the interaction between Alchemy replacing original modifiers and fractured modifiers being immutable.

No runtime, data semantics, operation admission, public numeric probability, planner, optimizer, automation, or open-boundary closure is included.
