# Gated Scope and M37-A Floor Proposal

## Recommended M37-A

Implement base Chaos-like `remove_then_add` runtime only:

- operation row: `chaos`;
- rare item only;
- no Omens;
- no Whittling;
- no Greater/Perfect MML modes;
- no route planner;
- no optimizer/advice/economics;
- no public numeric probability release.

## Why this floor is safe

Base Chaos-like runtime composes two already accepted behavior families:

- base Annulment-style removable-instance selection;
- accepted ordinary add pool building after branch-state mutation.

The new work is operation atomicity and branch-specific composition, not a new source-truth claim.

## Separately gated

- `greater_chaos` and `perfect_chaos` MML modes.
- Omen side filters.
- Whittling / minimum modifier-level selection.
- Any Chaos variants not explicitly admitted.
- Longer chains or route planner.
- Public numeric release.
- Optimizer/economics/advice.
- SOURCE/PROVENANCE, MML, PD-013 closure.

## Acceptance criteria for later M37-A

- fail-closed runtime admission;
- no fractured removal;
- no partial remove-only commit;
- branch-specific add-pool rebuild;
- exact rational path enumeration;
- canonical terminal aggregation;
- exact mass conservation;
- seeded MC replay;
- diagnostics and negative controls;
- accepted single-operation behavior remains unchanged.
