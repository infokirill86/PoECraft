# Participant and Architecture Critique

## Material correction to the implied framing

The next wave should not default to another operation or modifier admission.

That would have been correct before M40–M42. It is no longer the highest-value move. P2C now has accepted rarity progression, Exalted, Annulment, Chaos-like, Greater Essence, and Perfect Essence behavior, including their admitted variants. The operation surface is mature enough; the product bottleneck has moved to composition.

## Why bounded sequences are not infrastructure drift

This layer is the direct bridge from “a collection of currency emulators” to “a crafting-route simulator.” A fixed user-authored sequence is a product capability:

- each later operation sees the real branch state produced earlier;
- rarity, capacity, family/group blocking, fractured protection, and operation preconditions are re-evaluated at every step;
- exact and Monte Carlo execution can evaluate the same route without choosing it.

## Correct boundary

M43 should design a bounded fixed-sequence evaluator over the accepted runtime registry.

It must not become:

- a route planner;
- a conditional policy language;
- a retry loop;
- a ranking or recommendation system;
- an economics layer;
- a path for bypassing `runtime_admission_status`.
