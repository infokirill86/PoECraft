# Selected M43 Boundary and Route Connection

## Selected boundary

M43 is **bounded fixed-sequence evaluation over already accepted operations**.

Input shape:

```yaml
sequence_id: user_supplied_identifier
stop_on_no_transition: true
steps:
  - step_id: stable_unique_id
    currency_id: an_accepted_operation_id
    mode_id: stable_run_label
```

Catalog variants remain independent admitted currency rows. `active_modifier_ids` must remain empty until a later Omen gate.

## Direct connection to the intended route

The primary active start is a Rare quarterstaff with a required fractured critical suffix. M43 can evaluate a user-authored sequence such as an accepted Essence action followed by accepted Annulment, Chaos-like, or Exalted actions, while rebuilding legality from the actual branch state at each step.

This does not prescribe that example as a recommended strategy. It proves the engine can execute the route the user supplies.

## Proposed first implementation bound

M43-A should accept fixed sequences from one through eight steps. The limit is a safety ceiling for the first implementation, not a game rule and not a permanent route-length claim.

Exact enumeration is required only while pinned path/terminal ceilings are not exceeded. Seeded MC and invariant checking remain available throughout the eight-step bound.

## Explicit exclusions

- no branching on intermediate item quality;
- no “repeat until” behavior;
- no fallback action;
- no route discovery or comparison;
- no cost or expected-attempt calculation;
- no Omens or new operation admissions;
- no public probability values.
