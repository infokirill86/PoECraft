# Clean Core vs Disputed Edges

| Surface | Clean M46-A proposal | Separate gate |
|---|---|---|
| Rare quarterstaff, four or more ordinary/crafted explicits | include | no |
| Existing fractured modifier | fail closed | multiple-fracture behavior remains closed |
| Ordinary explicit modifier candidate | include uniformly by instance identity | no |
| Crafted/Essence explicit candidate | include; preserve `crafted` flag; source-aligned project model | broader crafted-capacity/repeat contract remains open |
| Revealed Desecrated modifier | reject entire input in M46-A | yes: PD-013/source conflict |
| Unrevealed Desecrated placeholder | reject entire input in M46-A | yes: Jawbone/Reveal contour |
| Side-directed Fracture | unsupported | yes, if any verified modifier layer appears later |
| Non-quarterstaff base | outside M46-A | later generalization gate |
| Unique/corrupted/special item rules | unsupported unless already represented and separately admitted | later source/mechanics gate |

This separation prevents disputed special-state behavior from contaminating the clean core while preserving a useful, auditable Fracture operation for the intended route.
